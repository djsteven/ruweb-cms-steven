# Image Strategy

## Purpose

This guide defines the shared image handling strategy for the starter:

- upload-time optimization
- responsive variant generation
- traceability of optimization results
- frontend rendering expectations
- operational health checks

## Goals

- optimize eligible raster uploads
- preserve measurable before/after metadata
- generate responsive variants for supported images
- expose health signals for stored media

## Configuration Areas

Typical configuration includes:

- image optimization enabled/disabled
- maximum raster width
- quality level
- whether originals are preserved
- responsive image widths

Keep the exact key names in configuration, but avoid duplicating full config reference tables across multiple docs unless this file is the chosen source of truth.

## Upload Pipeline

When an eligible image is uploaded, the pipeline typically:

1. records original metadata
2. optimizes the image to the target format
3. updates stored file metadata
4. generates responsive variants
5. optionally preserves or removes the original file

Non-raster assets such as SVGs and generic documents should bypass raster optimization and responsive variant generation.

The default pipeline optimizes `jpg`, `jpeg`, and `png` uploads to WebP. It generates responsive variants for `jpg`, `jpeg`, `png`, and `webp`. GIFs can have dimensions read as raster images, but they are not converted or variant-generated by default.

## Traceability

Optimization should remain measurable per asset.

Typical fields include:

- final size
- original size
- original format metadata
- original preserved path, when applicable
- width and height
- variant metadata
- optimization timestamp

## Rendering Contract

Public templates should prefer a shared image component or helper that:

- renders `srcset` when variants exist
- falls back to a standard `src` when variants do not exist
- resolves alt text predictably

Avoid reimplementing responsive image logic inside individual templates.

## Operations

Typical commands:

```bash
php artisan media:convert-webp
php artisan media:generate-variants
php artisan media:audit-health
```

Guideline:

- run conversion before variant generation
- use dry-run or force options intentionally

## Health Reporting

Useful operational metrics include:

- total media assets
- optimizable images
- optimized coverage
- responsive variant coverage
- bytes saved
- missing physical files
- assets still pending optimization

## Scope Boundary

This document should not become:

- a deployment guide
- a duplicate of the media maintenance command list
- a file-by-file implementation walkthrough
